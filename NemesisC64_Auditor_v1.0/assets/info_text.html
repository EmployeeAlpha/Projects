NemesisC64 Auditor

Version 1.0 · WPF + Python Bridge

What is this?

A desktop auditor that orchestrates quick searches and collections via a Python bridge. The WPF app calls python/runner.py, which can leverage an embedded vendor engine at python/vendor/robin/ if present. Results appear as a summary, findings list, and a saved text report.

Quick Start

  1. Click Run Audit.
  2. (Optional) Enter a Quick search query, a Webhook URL, or Email fallback.
  3. Watch Live Output and review Findings.
  4. Use Save Report… or Open Reports to view generated files.

Folders

Reports
/reports/ (timestamped .txt reports)
Logs
/logs/ (request/response JSON + stdout/stderr)
Assets
/assets/ (footer ASCII & this page)
Python
/python/ (bridge & vendor engines)

How it works (high level)

[WPF UI] ──► creates input JSON
    │
    ├─► launches Python:  python/runner.py --input ... --output ...
    │
    ├─► runner.py tries vendor engine at python/vendor/robin/
    │      └─ if found: runs search/scrape to collect findings
    │      └─ else : returns a safe placeholder result
    │
    ├─► runner.py writes output JSON + /reports/auditor_report_*.txt
    │
    └─► WPF reads JSON, updates Live Output + Findings grid

Webhook & Email

  • Webhook: If a URL is provided, runner tries a JSON POST (best-effort). Header X-Auditor-Secret is sent when a secret is set.
  • Email: UI fields are placeholders for future SMTP integration. For now, export or save reports locally.

Notes & Safety

  • Local-only by default; no network calls unless the vendor engine or webhook is used.
  • Limit findings to what you intend to process. Respect privacy, laws, and terms of use.
  • Tor checkbox signals vendor modules that a Tor-friendly path is preferred (if supported).
